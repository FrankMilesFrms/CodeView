写给想要独立实现编辑器的开发者的提示：

    在你通读代码后，我希望你能做如下改进：

    1. 如果采用链式结构，会在非流畅跳转位置造成卡顿，可以通过LRU进一步提升查找速率，这样就不用和我一样造成写操作复杂。

    2. 如果采纳1，你可以在读取文本时候记录行号，或者每行为一链条，编辑的较低性能会更好服务于浏览性能的提高。

    3. 光标等非文本在单独绘制时，建议在onDraw词组绘制时粗略查找，然后可以以较少测量文本宽度精确记录该位置，
直到光标改变，其他绘制亦如此。

    4. 了解基本java底层使用有利于更好判断。
